You are an expert systems engineer using the Arcadia/Capella methodology. Your task is to generate a Capella Logical Architecture (LA) layer from a set of system requirements.

The Logical Architecture opens the system black box into logical subsystems (components). You define the logical decomposition of system functions onto components, without committing to specific hardware technology.

Key Arcadia LA concepts:
- Logical Components (LC): Functional subsystems of the vessel (e.g., Navigation System, Communication System, Propulsion System, Deck Operations System, Monitoring System)
- Logical Functions (LF): Functions allocated to a specific logical component (e.g., Compute Position allocated to Navigation System)

For a PIB Icebreaker the typical logical components include: Navigation & Station-Keeping System, Communication & GMDSS System, Propulsion & Manoeuvring System, Deck & Launch/Recovery System, Bridge & Command System, Environmental Monitoring System.

---

INPUT

The following requirements are provided as a JSON array:

{requirements}

---

EXAMPLE OUTPUT SCHEMA

{{
  "components": [
    {{
      "id": "LC-001",
      "name": "Navigation & Station-Keeping System",
      "functions": ["LF-001", "LF-002"]
    }},
    {{
      "id": "LC-002",
      "name": "Communication & GMDSS System",
      "functions": ["LF-003"]
    }}
  ],
  "functions": [
    {{
      "id": "LF-001",
      "name": "Compute Vessel Position",
      "component": "LC-001"
    }},
    {{
      "id": "LF-002",
      "name": "Execute Station-Keeping Control",
      "component": "LC-001"
    }},
    {{
      "id": "LF-003",
      "name": "Monitor GMDSS Channels",
      "component": "LC-002"
    }}
  ]
}}

---

OUTPUT

Generate a JSON object matching the schema above, derived from the provided requirements. Return ONLY the JSON object with no additional explanation, preamble, or markdown formatting.

Rules:
1. Every function ID listed inside a component's "functions" array MUST have a corresponding entry in the top-level "functions" list.
2. The "component" field in each function MUST reference a valid component ID.
3. Component IDs follow LC-NNN, function IDs follow LF-NNN.
4. Allocate functions to components based on physical and functional affinity.
5. Each requirement should map to at least one logical function.
